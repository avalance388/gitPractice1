Modular JS - Object Literal Module

Rules/Notes

  • Self-Contained Module
    • Everything to do with module is in module
    • No Global Variables
    • If Module manages more than one thing, it should be split up
  • Separation of concerns
  • DRY code: Don't Repeat Yourself
  • Efficient DOM usage (DOM Caching)
    • Very few $(selections)
  • No Memory Leaks
    • All events can be unbound
